# 超声波测距系统 - 项目架构设计

## 1. 系统概述

基于 STM32F103C8T6 的超声波测距系统，实现距离测量、OLED 显示、语音播报、蜂鸣器报警功能。

| 项目       | 说明                          |
| ---------- | ----------------------------- |
| 主控       | STM32F103C8T6 (72MHz, 64K Flash, 20K RAM) |
| 测距模块   | HC-SR04 (2cm~400cm, 精度1cm)  |
| 显示模块   | 0.96寸 OLED (I2C, SSD1306)   |
| 语音模块   | SYN6288 (UART)                |
| 报警模块   | 有源蜂鸣器 + NPN三极管驱动    |
| 开发环境   | CubeMX 生成 Keil5 工程 + HAL库 |

## 2. 硬件架构

### 2.1 系统框图

```
┌─────────────────────────────────────────────────────┐
│                  STM32F103C8T6                       │
│                                                      │
│  HC-SR04 ── TRIG ──── PA0 (GPIO Output)            │
│  HC-SR04 ── ECHO ──── PA1 (TIM2_CH2 Input Capture) │
│                                                      │
│  OLED ─── SCL ─────── PB6 (I2C1_SCL)               │
│  OLED ─── SDA ─────── PB7 (I2C1_SDA)               │
│                                                      │
│  SYN6288 ── RXD ───── PA2 (USART2_TX)              │
│  SYN6288 ── TXD ───── PA3 (USART2_RX)  [可选]      │
│  SYN6288 ── BUSY ──── PA4 (GPIO Input)             │
│                                                      │
│  蜂鸣器 ── SIG ─────── PB0 (GPIO Output)            │
│  按键 ─────────────── PB13 (GPIO Input, 阈值调节)   │
│                                                      │
│  USART1 ─────────── PA9(TX) / PA10(RX)  调试串口    │
└─────────────────────────────────────────────────────┘
```

### 2.2 详细引脚分配表

| 外设     | 引脚  | 功能             | 说明                     |
| -------- | ----- | ---------------- | ------------------------ |
| HC-SR04  | PA0   | TRIG (输出)      | 10μs以上高脉冲触发测距   |
| HC-SR04  | PA1   | ECHO (输入)      | TIM2 CH2 输入捕获，测回响脉宽 |
| OLED     | PB6   | I2C1_SCL         | 时钟线                   |
| OLED     | PB7   | I2C1_SDA         | 数据线                   |
| SYN6288  | PA2   | USART2_TX        | 发送语音合成指令 (通RXD) |
| SYN6288  | PA3   | USART2_RX        | 接收模块反馈 (通TXD,可选)|
| SYN6288  | PA4   | GPIO_IN (上拉)   | BUSY 忙信号检测          |
| 蜂鸣器   | PB0   | GPIO_OUT (推挽)  | SIG型模块: HIGH=响       |
| 按键     | PB13  | GPIO_IN (上拉)   | 长短按调阈值             |
| 调试串口 | PA9   | USART1_TX        | 调试信息输出 (115200)    |
| 调试串口 | PA10  | USART1_RX        | 调试指令输入             |

### 2.3 供电方案

```
USB 5V (或锂电池 3.7V+升压)
    │
    ├──→ 5V ───→ HC-SR04 VCC
    ├──→ 5V ───→ SYN6288 VCC (需5V供电，电流≥200mA)
    ├──→ 5V ──┬→ AMS1117-3.3 → 3.3V ──→ STM32 VDD
    │         │                       ├→ OLED VCC
    │         │                       └→ 蜂鸣器驱动
    │         └→ 滤波电容: 100μF电解 + 0.1μF瓷片
    └──→ 共地 GND
```

**注意事项：**
- SYN6288 工作电压 3.6V~5V，推荐 5V 供电，电流需求约 200mA（播音时）
- HC-SR04 工作电压 5V，但 ECHO 输出 5V 电平，STM32 的 IO 是 5V 耐受的，可直连
- OLED 0.96寸 I2C 版通常自带 3.3V 稳压，可接 3.3V 或 5V

### 3.3 HC-SR04 驱动设计

**测距原理：**
- TRIG 引脚发送 ≥10μs 高电平触发信号
- 模块自动发送 8 个 40kHz 方波
- ECHO 引脚输出高电平，持续时间为超声波往返时间
- 距离 = (高电平时间 × 340m/s) / 2

**定时器配置方案：**
- PA0 作为 TRIG GPIO 输出，软件发送脉冲
- 使用 **TIM2** 的 **CH2** (PA1) 输入捕获测量 ECHO 脉宽
- TIM2 预分频 71 (1MHz计数)，自动重装载 65535
- 每个计数 = 1μs，距离(cm) = 计数值 / 58

**API 设计：**
```c
void  HCSR04_Init(void);           // 初始化
void  HCSR04_Start(void);          // 触发一次测距
float HCSR04_GetDistance(void);    // 获取距离(厘米)
uint8_t HCSR04_IsDone(void);       // 是否完成测量（超时返回旧值判断）
```

### 3.4 OLED 驱动设计

- 0.96寸 SSD1306 I2C 接口，7位地址 0x3C (HAL层左移后为 0x78)
- 核心驱动文件：ssd1306.c / ssd1306.h
- 架构：直接写 GDDRAM（无帧缓冲），对照野火 `bsp_i2c_oled.c` 方案
- 显示字体：6×8 和 8×16 ASCII

**API 设计：**
```c
void SSD1306_Init(void);
void SSD1306_ClearBuffer(void);
void SSD1306_UpdateScreen(void);  // 空函数，无帧缓冲无需刷新
void SSD1306_ShowString_6x8(uint8_t x, uint8_t y, const char *str, uint8_t color);
void SSD1306_ShowNum(uint8_t x, uint8_t y, int32_t num, uint8_t len, uint8_t font_size);
```

### 3.5 SYN6288 语音驱动设计

**通信协议：**
- UART 波特率：9600bps (对照官方参考项目)
- 数据格式：帧头(0xFD) + 数据长度 + 编码/命令 + 文本 + XOR校验

**文本合成帧格式（GB2312编码）：**
```
FD 00 LL 01 01 [GB2312文本] [XOR]
         ^^ ^^
         │  └─ 合成命令 = 0x01 | (Music<<4)
         └── LL = 文本字节数 + 3
XOR 范围：从 Frame[0]（包含FD）到最后一个文本字节
```
对照参考项目 `\7.1、STM32F103C8T6应用程序\` 中的 `SYN_FrameInfo()`。

**芯片控制帧格式（停止/暂停/恢复）：**
```
FD 00 02 CMD XOR     (5字节)
XOR = FD ^ 00 ^ 02 ^ CMD
```

**API 设计：**
```c
void SYN6288_Init(void);
void SYN6288_SpeakDistance(uint8_t distance_cm); // 播报"当前距离XX厘米"
void SYN6288_SendText(const uint8_t *text, uint8_t len);
void SYN6288_SetVolume(uint8_t vol);     // 音量0-15
uint8_t SYN6288_IsBusy(void);            // 检查是否正在播报
```

**播报策略：**
- 每秒检测一次，距离变化超过5cm或低于阈值时播报
- 正常状态每3秒播报一次
- 报警状态每1秒播报一次

### 3.6 蜂鸣器驱动设计

```c
void Buzzer_Init(void);
void Buzzer_Beep(uint16_t freq_ms);     // 指定频率的滴声
void Buzzer_Alarm(uint8_t enable);      // 急促报警音 (100ms间隔)

```

## 4. CubeMX 配置指南

### 4.1 新建工程

1. 打开 STM32CubeMX
2. New Project → 选择 **STM32F103C8Tx**
3. 进入 Pinout & Configuration

### 4.2 时钟配置 (Clock Configuration)

```
HSE: 8MHz (外部晶振, Crystal/Ceramic Resonator)
PLL Source: HSE
PLL Mul: ×9 → 72MHz
SYSCLK: 72MHz
HCLK: 72MHz
APB1: 36MHz (max)
APB2: 72MHz (max)
```

### 4.3 外设配置 (Pinout)

| 引脚 | 模式                                  | 参数配置                                |
| ---- | ------------------------------------- | --------------------------------------- |
| PA0  | GPIO_Output (推挽)                    | TRIG 脉冲输出, Low Speed                |
| PA1  | TIM2_CH2 (Input Capture direct)       | Channel2: Rising→Falling, Prescaler=71, ARR=65535 |
| PB6  | I2C1_SCL                              | Speed: Fast Mode (400kHz)               |
| PB7  | I2C1_SDA                              | 同上                                    |
| PA2  | USART2_TX                             | Baud: 9600, 8N1                         |
| PA3  | USART2_RX                             | 同上                                    |
| PA4  | GPIO_Input                            | Pull-up (SYN6288 BUSY)                  |
| PB0  | GPIO_Output (推挽)                    | 蜂鸣器 SIG, Low Speed                   |
| PB13 | GPIO_Input                            | Pull-up (按键)                          |
| PA9  | USART1_TX                             | Baud: 115200, 8N1                       |
| PA10 | USART1_RX                             | 同上                                    |

### 4.4 项目设置 (Project Manager)

- Project Name: `UltrasonicRanger`
- Toolchain/IDE: **MDK-ARM V5** (Keil5)
- Heap/Stack: 默认即可
- Code Generator:
  - [x] Copy only the necessary library files
  - [x] Generate peripheral initialization as a pair of '.c/.h' files
