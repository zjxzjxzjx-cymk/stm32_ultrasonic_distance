### 2. 面包板搭建步骤

#### 步骤 1：电源布局

```
将面包板上下电源轨分配合：
  上轨：5V (红线)
  下轨：3.3V (橙线)
  两侧蓝线：GND (黑线)

连接方式：
  USB 5V → 面包板上轨 5V
  USB GND → 面包板 GND 轨

  （STM32最小系统板已有3.3V稳压，可从板子3.3V引脚引出
   到面包板下轨给OLED供电）
```

**实际操作：**

1. 将 STM32 最小系统板插入面包板中间区域（跨中间凹槽）
2. 用杜邦线连接：
   - 面包板 5V 轨 → STM32 板的 `5V` 引脚
   - 面包板 GND 轨 → STM32 板的 `G` 引脚
   - STM32 板的 `3.3V` 引脚 → 面包板 3.3V 轨
3. 蓝板自带滤波电容，无需外接

#### 步骤 2：HC-SR04 接线

```
HC-SR04          STM32F103C8T6
┌──────┐         ┌──────────┐
│ VCC  │────────→│ 5V       │
│ GND  │────────→│ G        │
│ TRIG │────────→│ PA0      │
│ ECHO │────────→│ PA1      │
└──────┘         └──────────┘
```

**验证方法：** 
- 先用万用表确认 5V/GND 连接正确，无短路
- 上电后 HC-SR04 模块无异常发热

#### 步骤 3：OLED 接线

```
OLED (4Pin)      STM32F103C8T6
┌──────┐         ┌──────────┐
│ VCC  │────────→│ 3.3V     │
│ GND  │────────→│ G        │
│ SCL  │────────→│ PB6      │
│ SDA  │────────→│ PB7      │
└──────┘         └──────────┘
```

#### 步骤 4：SYN6288 接线

```
SYN6288模块                     STM32F103C8T6
┌──────────────┐               ┌──────────┐
│ VCC (5V)     │──────────────→│ 5V       │   注意：必须5V供电！
│ GND          │──────────────→│ G        │   ← 共地必须接！
│ RXD (模块收) │──────────────→│ PA2 TX   │   交叉连接
│ TXD (模块发) │──────────────→│ PA3 RX   │   交叉连接 (可选)
│ BUSY (忙信号)│──────────────→│ PA4      │   忙信号检测
├──────────────┤
│ SPK+/SPK-    │────→ 喇叭     │          │   模块自带功放输出
└──────────────┘
```

**注意事项：**
- SYN6288 的 RXD 接 STM32 的 TX（PA2），TXD 接 STM32 的 RX（PA3）——**交叉连接**
- **GND 必须共地**——STM32 的 GND 必须接到 SYN6288 的 GND，否则模块收不到信号
- 如果不使用 BUSY 和 TX 反馈，可以不接 PA3 和 PA4
- 喇叭插在 SYN6288 模块的 SPK+/SPK-，不是插在 STM32
- SYN6288 模块 LED 亮 = 供电正常

#### 步骤 5：蜂鸣器接线（4脚 SIG 模块）

```
蜂鸣器模块(4脚)        STM32
┌────────────┐      ┌──────────┐
│  3V3 (电源) │─────→│ 3.3V     │   供电
│  GND (接地) │─────→│ G        │   共地
│  SIG (信号) │─────→│ PB0      │   HIGH=响, LOW=不响
│  NC  (空脚) │      │ (不接)   │   空
└────────────┘      └──────────┘
```

**接线说明：**
- 模块内部自带三极管驱动，SIG 脚只需要 3.3V 逻辑电平，几乎不耗电流
- SIG=高电平 → 响，SIG=低电平 → 不响
- NC 脚不接任何东西

> `buzzer.h` 已配置 `BUZZER_DRIVE_HIGH` 模式（SIG=高→响）。

#### 步骤 6：按键接线（无需外部电阻）

```
PB12 ──→ 按键+ ──→ GND
PB13 ──→ 按键- ──→ GND
```

> PB12/PB13 已由 CubeMX 配置为内部上拉（40KΩ），不按=读到高电平，按下=读到低电平。
> 只需要按键和两根线，不需要任何电阻。PB12 按一次阈值+5cm，PB13 按一次阈值-5cm。

#### 步骤 7：ST-Link 接线

```
ST-Link V2        STM32F103C8T6
┌──────────┐      ┌──────────┐
│ SWCLK    │─────→│ SWCLK    │  (板上标注 SWCLK)
│ SWDIO    │─────→│ SWDIO    │  (板上标注 SWDIO)
│ GND      │─────→│ G        │
│ 3.3V     │─────→│ 3.3V     │  (可选，目标板自供电可省略)
└──────────┘      └──────────┘
```

### 3. 上电前的安全检查

- [ ] 用万用表蜂鸣档测量 5V 和 GND 是否短路
- [ ] 用万用表蜂鸣档测量 3.3V 和 GND 是否短路
- [ ] 确认所有 VCC/GND 极性正确
- [ ] 确认 SYN6288 的 RXD/TXD 是交叉连接（不是直连）
- [ ] 确认 HC-SR04 的 TRIG/ECHO 没有接反

### 4. 分模块逐步测试

#### 4.1 第一步：最小系统验证

1. CubeMX 生成 Keil 工程（参照《架构设计》第4节配置）
2. 只配置 GPIO（PA0, PA1, PB0, PB6, PB7, PA2, PA3, PB12, PB13）为输出
3. 编写简单的 LED 闪烁测试：
   ```c
   while (1) {
       HAL_GPIO_TogglePin(GPIOB, GPIO_PIN_0);
       HAL_Delay(500);
   }
   ```
4. ST-Link 烧录，验证程序能正常运行
5. 使用串口助手（115200bps）验证 USART1 printf 重定向正常输出

#### 4.2 第二步：HC-SR04 单独测试

1. TIM2 配置输入捕获（参考架构设计 3.3 节）
2. 编写测距测试代码，通过串口打印距离值
3. 用尺子在不同距离（5cm, 10cm, 20cm, 50cm, 100cm, 200cm）验证
4. 确认精度在 ±1cm 以内

**测试用例：**
| 实际距离 | 期望读数      | 允许误差  | 结果 |
| -------- | ------------- | --------- | ---- |
| 5cm      | 4~6cm         | ±1cm     |      |
| 20cm     | 19~21cm       | ±1cm     |      |
| 50cm     | 49~51cm       | ±1cm     |      |
| 100cm    | 99~101cm      | ±1cm     |      |
| 200cm    | 198~202cm     | ±2cm     |      |
| 超出范围 | 超时(-1或400) | --       |      |

#### 4.3 第三步：OLED 单独测试

1. 移植 ssd1306 OLED 驱动到工程
2. 编写测试：显示 "Hello World"、"距离: xxx cm"
3. 检查显示是否清晰，无闪烁

**常用 SSD1306 驱动库：** 可从 GitHub 搜索 `stm32 ssd1306 i2c` 参考移植，核心只有约 200 行代码。

#### 4.4 第四步：SYN6288 单独测试

1. USART2 发送语音合成指令
2. 编写测试代码（正确帧格式，对照官方参考项目）：
   ```c
   // 发送 "当前距离30厘米"
   // 帧格式：FD 00 LL 01 01 [文本] [XOR]
   // LL = 14(文本) + 3 = 17 = 0x11
   // XOR = FD^00^11^01^01^B5^B1^C7^B0^BE^E0^C0^EB^33^30^C0^E5^C3^D7
   uint8_t cmd[] = {0xFD, 0x00, 0x11, 0x01, 0x01,
      0xB5, 0xB1, 0xC7, 0xB0, 0xBE, 0xE0, 0xC0, 0xEB,
      0x33, 0x30, 0xC0, 0xE5, 0xC3, 0xD7, 0xDB};  // 共20字节
   HAL_UART_Transmit(&huart2, cmd, sizeof(cmd), 1000);
   ```
3. 听喇叭是否能正常播报
4. 确认 BUSY 信号时序正确

**上位机工具验证（可选）：**
在连接 SYN6288 之前，先用 USB-TTL 连接到电脑，用串口助手发送 HEX 指令测试模块是否正常。

#### 4.5 第五步：蜂鸣器测试

```c
// SIG 型模块：HIGH=响, LOW=停
HAL_GPIO_WritePin(GPIOB, GPIO_PIN_0, GPIO_PIN_SET);   // 长响
HAL_Delay(1000);
HAL_GPIO_WritePin(GPIOB, GPIO_PIN_0, GPIO_PIN_RESET); // 停
HAL_Delay(500);

// 滴滴报警测试
for (int i = 0; i < 5; i++) {
    HAL_GPIO_TogglePin(GPIOB, GPIO_PIN_0);
    HAL_Delay(100);
}
HAL_GPIO_WritePin(GPIOB, GPIO_PIN_0, GPIO_PIN_RESET); // 关
```

#### 4.6 第六步：全功能联调

2. 实现完整的主循环逻辑（参考架构设计 3.2 节）
3. 功能测试清单：

| 功能               | 测试方法                           | 预期结果                | 通过 |
| ------------------ | ---------------------------------- | ----------------------- | ---- |
| 距离测量           | 移动障碍物                         | OLED显示实时变化        |      |
| OLED显示           | 目视检查                           | 距离、阈值、状态清晰    |      |
| 语音播报           | 耳朵听取                           | 清晰播报"当前距离XX厘米"|      |
| 超阈值蜂鸣器       | 障碍物移近至<20cm                  | 急促滴滴声              |      |
| 正常范围不报警     | 障碍物移远至>20cm                  | 蜂鸣器停止              |      |
| 按键调阈值        | 按PB12                        | 阈值+5cm                |      |
|                    | 按PB13                        | 阈值-5cm                |      |

## 第二部分：PCB制版步骤

### 阶段一：原理图设计（使用立创EDA / Altium / KiCad）

推荐使用 **立创EDA（专业版）**，国产免费，元件库丰富。

#### 1.1 新建工程

- 工程名：`UltrasonicRanger_PCB`
- 原理图纸：A4 横版

#### 1.2 绘制原理图（分模块）

**原理图各模块：**

```
┌─────────────────────────────────┐
│ 1. MCU最小系统                  │
│    STM32F103C8T6 + 晶振 + 复位  │
│    去耦电容: 每个VDD引脚 0.1μF  │

│    4Pin排针/排母: VCC GND SCL SDA│

│    喇叭座: 2Pin 2.54mm          │
├─────────────────────────────────┤
│ 7. 蜂鸣器驱动                   │
```

### 阶段二：PCB Layout

#### 2.1 板框与布局规划

**推荐尺寸：** 80mm × 60mm 或 100mm × 70mm（双面板，2层）

**布局原则：**
```
┌─────────────────────────────────────┐
│  [SYN6288接口]  [喇叭座] [蜂鸣器]   │  ← 音频区（远离敏感信号）
│                                      │
│  [OLED接口]  [按键1] [按键2]        │  ← 人机交互区（板边）
│                                      │
│  ┌────────────────────┐             │
│  │  STM32F103C8T6     │             │  ← 中心位置
│  │  + 晶振 + 复位     │             │
│  └────────────────────┘             │
│                                      │
│  [电源模块 5V→3.3V]  [SWD下载口]    │
│                                      │
│  [HC-SR04接口]  [调试串口]          │  ← 传感器&调试（板边）
└─────────────────────────────────────┘
```

#### 2.2 布线规则

| 参数           | 值               | 说明                |
| -------------- | ---------------- | ------------------- |
| 最小线宽       | 10mil (0.254mm)  | 普通信号线          |
| 最小间距       | 8mil (0.2mm)     | 线到线/线到焊盘     |
| 电源线宽       | 20~30mil         | 5V和3.3V电源线      |
| GND            | 覆铜(铺铜)       | 大面积接地          |
| 过孔           | 内径12mil 外径24mil | 信号过孔          |
| 电源过孔       | 内径20mil 外径35mil | 电源过孔          |

#### 2.3 Layout 关键要点

1. **去耦电容放置：** 每个 VDD 引脚旁边紧贴一颗 0.1μF 电容，越近越好
2. **晶振布局：** 8MHz 晶振紧靠 OSC_IN/OSC_OUT 引脚，下方不走其他信号线，周围包地
3. **模拟和数字分区：** VDDA 引脚走线独立，通过磁珠/0Ω连接到 3.3V
4. **SYN6288 供电：** 5V 走线加粗（30mil+），模块旁边放 100μF 电容
5. **蜂鸣器：** 远离模拟信号和晶振区域
6. **接地：** 双面覆铜，多打过孔连接顶层和底层地，减少地阻抗
7. **接插件方向：** HC-SR04、OLED、SYN6288 的排母放在板边，方便插拔

#### 2.4 DRC 检查

- 电气规则检查
- 间距检查
- 丝印重叠检查

### 阶段三：打样与焊接

#### 3.1 导出 Gerber 文件

立创 EDA：文件 → 生成 Gerber → 一键导出
- 需要的层：GTL, GBL, GTS, GBS, GTO, GBO, GKO, TXT(钻孔)

#### 3.2 下单打样（以嘉立创为例）

1. 登录 jlc.com
2. 上传 Gerber 压缩包
3. 参数选择：
   - 板厚：1.6mm
   - 颜色：绿色（默认，最快）
   - 表面处理：有铅喷锡（最便宜）/ 无铅沉金
   - 铜厚：1oz
   - 数量：5片（嘉立创常规优惠）
4. 审单 → 付款 → 等待3~5天

#### 3.3 焊接顺序

```
1. 电源模块 → 上电测电压 (5V和3.3V是否正常)
       ↓ 正常
2. STM32 + 晶振 + 复位 → 烧录测试程序验证
       ↓ 能烧录
3. 去耦电容、排针排母等小元件
       ↓
4. HC-SR04 接口 → 测距验证
       ↓
5. OLED 接口 → 显示验证
       ↓
6. SYN6288 接口 + 喇叭座 → 语音验证
       ↓
7. 蜂鸣器驱动电路 → 报警验证
       ↓
8. 按键 → 全功能联调
```

**每焊完一个模块就测试，不要全部焊完再测试！**

## 附录：常见问题排查

| 现象                   | 可能原因                     | 解决方案                      |
| ---------------------- | ---------------------------- | ----------------------------- |
| HC-SR04 读数不变       | TRIG/ECHO 接反              | 交换接线                      |
| HC-SR04 读数跳动大     | 电源不稳 / 被测面不平        | 加电容滤波 / 对准平整墙面     |
| OLED 不显示            | I2C地址不对 / 未上拉          | 查地址（0x78还是0x7A）        |
| SYN6288 不播报         | 供电不足 / 波特率不对         | 串口助手检查 / 确认5V供电     |
| 蜂鸣器不响             | 用了无源蜂鸣器 / 三极管接错脚 | 必须用有源蜂鸣器 / 查S8050引脚|
| ST-Link 连不上         | SWD线序不对 / 复位脚问题     | 检查SWCLK/SWDIO/GND连接       |
| 程序烧录后不运行       | BOOT0 未接地                  | 检查 BOOT0=0, BOOT1=0         |
| 所有读数异常           | 晶振没起振 / 时钟配置错       | 示波器查晶振 / 重新配CubeMX   |
